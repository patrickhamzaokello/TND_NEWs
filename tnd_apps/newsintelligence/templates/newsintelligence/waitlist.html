Frequently Asked Questions

Based on community feedback and questions from early users

How is this different from Google's AI Overview?

Google's AI Overview answers a single search query with a one-time summary. This is different in a few key ways:

  • Not query-based — this is a standing page for one story that keeps updating as new reports come in over days, not a one-off answer.
  • Source-confirmation counts — each fact shows how many outlets confirm it, rather than one blended summary.
  • Timeline, not a snapshot — you see how the story evolved (first reports → official response → follow-ups), not just the current state.
  • Curated for Ugandan outlets specifically — Monitor, Nile Post, NBS, Chimp Reports, etc. — not generic web results.

Google shows you an answer. This tracks a story.

Is this different from what BBC, CNN, or Al Jazeera do with live blogs on big stories?

Similar concept — an evolving timeline — but built differently:

  • Live blogs are written by one newsroom as their own reporters cover events. This aggregates reporting from multiple independent outlets covering the same story.
  • Cross-source confirmation — a live blog gives you one outlet's version of events. This shows how many separate outlets confirm each fact, so readers can gauge reliability.
  • Built for local coverage gaps — BBC/CNN can do this because they staff a story directly. This works for stories where no single outlet has full coverage, but several together do.
Why does the timeline show the most recent event first instead of the first event first?

Newest-first follows the news-feed convention (like Twitter or live blogs), so returning readers immediately see what's new without scrolling past what they've already read.

That said, for first-time readers coming to a story cold, chronological order (oldest → newest) reads more naturally as a story arc. This is on the roadmap — likely as a toggle so readers can choose either view.
